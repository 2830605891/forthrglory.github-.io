网鼎杯第一场青龙组web部分WP

By Anonymous; Published on 2020-05-11

  • WP

太菜了，盯了半天MISC，一道题没做出来，准备吃饭放web了，饭都没吃看题目去了，也只做出来了一道(丢人.jpg)

Web

trace

一开始不知道要干嘛，测试了下发现username有注入，然后开始盲注。题目有坑点，如果注册数达到20，会直接返回Wrong，row>20，因此大于二十之后需要重新分发容器，手工注入跑出来库名是ctf，但是information_schema被过滤，导致注不出来表名，查询下版本是5.5.62，也没有sys，一度卡在这里。后来想起来库名是ctf，于是盲猜表名是flag，还真猜对了，利用无列名注入跑flag即可

因为当时做的时候还剩下半个小时左右，所以时间非常紧张，也没有截图，脚本跑注入，手动分发容器，结果最后跑出来的时候还是晚了一步，没交上- -

附上脚本

第一个循环跑字符，第二个循环跑数字，没有-、{、}，因为当我跑出来前四个字符是flag的时候，我就猜格式和签到的flag格式一样，flag{xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx}，别问为啥猜，问就是赌狗:)

赛后复盘

AreUSerialz

本地起的环境，一开始是php5.4.45的，比赛的时候考虑过用public，直接报错，后来没再管。比赛完了，看大佬的wp才知道7.1以上对属性不敏感- -，有大佬说目录还有问题，但我本地测试直接读flag.php没问题，有时间学习下。

trace

赛后和其他队伍讨论了一下，他说那个结果可以绕过，只要让insert语句报错，然后执行自己的语句就可以了。。。心态直接炸裂

总结

总共四道题目，只做出来一道，另外的Java和js找不到漏洞点，还是tcl，借这次比赛找到了薄弱点，也算是有所收获，不虚此行。
